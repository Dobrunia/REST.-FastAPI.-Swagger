# Git — система контроля версий

## Что такое Git

Git — это распределённая система контроля версий, которая позволяет отслеживать изменения в файлах и координировать работу нескольких разработчиков над одним проектом. Git хранит полную историю изменений локально, что делает его быстрым и независимым от сети.

## Основные команды

### Инициализация и клонирование

```bash
git init                    # Создать новый репозиторий
git clone <url>             # Клонировать существующий репозиторий
```

### Работа с изменениями

```bash
git status                  # Показать состояние файлов
git add <file>              # Добавить файл в индекс
git add .                   # Добавить все изменения
git commit -m "message"     # Создать коммит с сообщением
git diff                    # Показать различия
```

### Работа с ветками

```bash
git branch                  # Список веток
git branch <name>           # Создать новую ветку
git checkout <branch>       # Переключиться на ветку
git checkout -b <branch>    # Создать и переключиться на ветку
git merge <branch>          # Слить ветку в текущую
git branch -d <branch>      # Удалить ветку
```

### Работа с удалённым репозиторием

```bash
git remote -v               # Показать удалённые репозитории
git push origin <branch>    # Отправить изменения
git pull origin <branch>    # Получить и слить изменения
git fetch                   # Получить изменения без слияния
```

## Лучшие практики

1. **Атомарные коммиты** — каждый коммит должен содержать одно логическое изменение
2. **Осмысленные сообщения** — пишите понятные commit messages
3. **Частые коммиты** — не накапливайте много изменений
4. **Feature branches** — разрабатывайте новые функции в отдельных ветках

## Частые вопросы

**Как отменить последний коммит?**

```bash
git reset --soft HEAD~1    # Сохранить изменения в индексе
git reset --hard HEAD~1    # Полностью удалить изменения
```

**Как посмотреть историю коммитов?**

```bash
git log --oneline          # Краткая история
git log --graph --all      # С визуализацией веток
```

**Как разрешить конфликт слияния?**

1. Откройте файлы с конфликтами
2. Выберите нужные изменения между `<<<<<<<` и `>>>>>>>`
3. Удалите маркеры конфликта
4. Сделайте `git add` и `git commit`
